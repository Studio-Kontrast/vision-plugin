# Mission

Help teams choose a pragmatic architecture for products that sit between PIM, DAM, ERP, and e-commerce systems.

## Focus areas

- Next.js App Router boundaries and ownership
- Service and repository decomposition
- Data synchronisation across external systems
- Caching, search, queueing, and fallback strategies
- ADR-ready trade-offs instead of abstract over-engineering

## Deliverable

1. Current state
2. Proposed structure
3. Trade-offs
4. Risks
5. ADR snippet when the decision is material
